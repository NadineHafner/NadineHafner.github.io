There are some very important questions about Usability!

  1. Why is Usability so important? In order to avoid a switchover to another company, frustrations with your Webpage need to be minimized:
    • hidden functions that are never used
    • the result of an action is not visible eg you click on a button and nothing happens
    • important information is very hard to find
    • the aim of your company is not clear
  2. What is a so - called "Norman Door"? An user has to guess about whether to push or pull a door, because there is no special signage how to use it.
  3. What are Affordances? relationship between the characteristics of an object and the abilities of a person, determining the usage of the object. There is a difference between:
    1. real Affordances: actions which are really possible
    2. perceived Affordances: actions the user thinks of being possible
  4. What are mappings? Relationships between the regulation of an object and its results on the object eg the swich - on of the hot - plates
  5. Name usability - attributes (Nielsen)! All are shown in picture:
    • Learnability: new users can learn the system fast
    • Efficiency: experienced users reach better performance eg. shortcuts etc.
    • Memorability: occasional users dont have to learn from the beginning everytime they use the system
    • Errors: Errors are failed actions according to their aim. In ideal circumstances the error rate is very small and errors can be solved very fast
    • Effectiveness: Precision and completeness of the reached aims of an action
    • Satisfaction: Every user should feel satisfied when using the system
  6. Heuristic Feedback: What does it mean and why is it so important? If the aim of an action is perceived only one second later, the System seems to react immediately. If the reaction lasts longer, Feedback eg a progress bar should be given as a sign that the System is not interrupted
  7. Heuristic Reversible Actions: Why is it important? When using a System, the user occasionally makes mistakes. For example, he might accidentally click on a button. There should be a possibility to undo mistakes easily
  8. Heuristic Error Prevention: What does it mean? A well designed System is able to avoid some mistakes of users by providing information what to do to reach an action. For example, when registering and typing a invalid telephone number, there is a warning that this number does not exist
  9. What is a Persona and why is it important in Heuristic Evaluation?A persona is a prototypical User of a Webpage. A Persona is defined with all his/her attributes, hopes and fears so that evaluators not only investigate frustrations of the general user but of a concrete fictive person

Keith Andrews uses soup - analogy for explaining different kinds of evaluation
